Verify that the end-entity certificate end_entity is a valid server cert and chains to at least one of the trust anchors in the roots RootCertStore.

This function is primarily useful when building a custom certificate verifier. It performs no revocation checking. Implementers must handle this themselves, along with checking that the server certificate is valid for the subject name being used (see verify_server_name).
