16//! A simple event-driven library for parsing WebAssembly binary files
17//! (or streams).
18//!
19//! The parser library reports events as they happen and only stores
20//! parsing information for a brief period of time, making it very fast
21//! and memory-efficient. The event-driven model, however, has some drawbacks.
22//! If you need random access to the entire WebAssembly data-structure,
23//! this is not the right library for you. You could however, build such
24//! a data-structure using this library.
25//!
26//! To get started, create a [`Parser`] using [`Parser::new`] and then follow
27//! the examples documented for [`Parser::parse`] or [`Parser::parse_all`].
28
29#![deny(missing_docs)]
30#![no_std]
31#![cfg_attr(docsrs, feature(doc_auto_cfg))]

56/// A helper macro to conveniently iterate over all opcodes recognized by this
57/// crate. This can be used to work with either the [`Operator`] enumeration or
58/// the [`VisitOperator`] trait if your use case uniformly handles all operators
59/// the same way.
60///
61/// It is also possible to specialize handling of operators depending on the
62/// Wasm proposal from which they are originating.
63///
64/// This is an "iterator macro" where this macro is invoked with the name of
65/// another macro, and then that macro is invoked with the list of all
66/// operators. An example invocation of this looks like:
67///
68/// The list of specializable Wasm proposals is as follows:
69///
70/// - `@mvp`: Denoting a Wasm operator from the initial Wasm MVP version.
71/// - `@exceptions`: [Wasm `exception-handling` proposal]
72/// - `@tail_call`: [Wasm `tail-calls` proposal]
73/// - `@reference_types`: [Wasm `reference-types` proposal]
74/// - `@sign_extension`: [Wasm `sign-extension-ops` proposal]
75/// - `@saturating_float_to_int`: [Wasm `non_trapping_float-to-int-conversions` proposal]
76/// - `@bulk_memory `:[Wasm `bulk-memory` proposal]
77/// - `@threads`: [Wasm `threads` proposal]
78/// - `@simd`: [Wasm `simd` proposal]
79/// - `@relaxed_simd`: [Wasm `relaxed-simd` proposal]
80/// - `@gc`: [Wasm `gc` proposal]
81///
82/// [Wasm `exception-handling` proposal]:
83/// https://github.com/WebAssembly/exception-handling
84///
85/// [Wasm `tail-calls` proposal]:
86/// https://github.com/WebAssembly/tail-call
87///
88/// [Wasm `reference-types` proposal]:
89/// https://github.com/WebAssembly/reference-types
90///
91/// [Wasm `sign-extension-ops` proposal]:
92/// https://github.com/WebAssembly/sign-extension-ops
93///
94/// [Wasm `non_trapping_float-to-int-conversions` proposal]:
95/// https://github.com/WebAssembly/nontrapping-float-to-int-conversions
96///
97/// [Wasm `bulk-memory` proposal]:
98/// https://github.com/WebAssembly/bulk-memory-operations
99///
100/// [Wasm `threads` proposal]:
101/// https://github.com/webassembly/threads
102///
103/// [Wasm `simd` proposal]:
104/// https://github.com/webassembly/simd
105///
106/// [Wasm `relaxed-simd` proposal]:
107/// https://github.com/WebAssembly/relaxed-simd
108///
109/// [Wasm `gc` proposal]:
110/// https://github.com/WebAssembly/gc
111///
112/// ```
113/// macro_rules! define_visit_operator {
114///     // The outer layer of repetition represents how all operators are
115///     // provided to the macro at the same time.
116///     //
117///     // The `$proposal` identifier indicates the Wasm proposals from which
118///     // the Wasm operator is originating.
119///     // For example to specialize the macro match arm for Wasm SIMD proposal
120///     // operators you could write `@simd` instead of `@$proposal:ident` to
121///     // only catch those operators.
122///     //
123///     // The `$op` name is bound to the `Operator` variant name. The
124///     // payload of the operator is optionally specified (the `$(...)?`
125///     // clause) since not all instructions have payloads. Within the payload
126///     // each argument is named and has its type specified.
127///     //
128///     // The `$visit` name is bound to the corresponding name in the
129///     // `VisitOperator` trait that this corresponds to.
130///     ($( @$proposal:ident $op:ident $({ $($arg:ident: $argty:ty),* })? => $visit:ident)*) => {
131///         $(
132///             fn $visit(&mut self $($(,$arg: $argty)*)?) {
133///                 // do nothing for this example
134///             }
135///         )*
136///     }
137/// }
138///
139/// pub struct VisitAndDoNothing;
140///
141/// impl<'a> wasmparser::VisitOperator<'a> for VisitAndDoNothing {
142///     type Output = ();
143///
144///     wasmparser::for_each_operator!(define_visit_operator);
145/// }
146/// ```
