15/// Trait for integrating a garbage collector with the runtime.
16///
17/// This trait is responsible for:
18///
19/// * GC barriers used by runtime code (as opposed to compiled Wasm code)
20///
21/// * Creating and managing GC heaps for individual stores
22///
23/// * Running garbage collection
24///
25/// # Safety
26///
27/// The collector, its GC heaps, and GC barriers when taken together as a whole
28/// must be safe. Additionally, they must work with the GC barriers emitted into
29/// compiled Wasm code via the collector's corresponding `GcCompiler`
30/// implementation. That is, if callers only call safe methods on this trait
31/// (while pairing it with its associated `GcCompiler`, `GcHeap`, and etc...)
32/// and uphold all the documented safety invariants of this trait's unsafe
33/// methods, then it must be impossible for callers to violate memory
34/// safety. Implementations of this trait may not add new safety invariants, not
35/// already documented in this trait's interface, that callers need to uphold.
36pub unsafe trait GcRuntime: 'static + Send + Sync {
37    /// Construct a new GC heap.
38    fn new_gc_heap(&self) -> Result<Box<dyn GcHeap>>;
39
40    /// Get this collector's layout for the given array type.
41    fn array_layout(&self, ty: &WasmArrayType) -> GcArrayLayout;
42
43    /// Get this collector's layout for the given struct type.
44    fn struct_layout(&self, ty: &WasmStructType) -> GcStructLayout;
45}

47/// A heap that manages garbage-collected objects.
48///
49/// Each `wasmtime::Store` is associated with a single `GcHeap`, and a `GcHeap`
50/// is only ever used with one store at a time, but `GcHeap`s may be reused with
51/// new stores after its original store is dropped. The `reset` method will be
52/// called in between each such reuse. (This reuse allows for better integration
53/// with the pooling allocator).
54///
55/// If a `GcHeap` mapped any memory, its `Drop` implementation should unmap that
56/// memory.
57///
58/// # Safety
59///
60/// The trait methods below are all safe: implementations of this trait must
61/// ensure that these methods cannot be misused to create memory unsafety. The
62/// expectation is that -- given that `VMGcRef` is a newtype over an index --
63/// implementations perform similar tricks as Wasm linear memory
64/// implementations. The heap should internally be a contiguous region of memory
65/// and `VMGcRef` indices into the heap must be bounds checked (explicitly or
66/// implicitly via virtual memory tricks).
67///
68/// Furthermore, if heap corruption occurs because (for example) a `VMGcRef`
69/// from a different heap is used with this heap, then that corruption must be
70/// limited to within this heap. Every heap is a mini sandbox. It follows that
71/// native pointers should never be written into or read out from the GC heap,
72/// since that could spread corruption from inside the GC heap out to the native
73/// host heap. The host data for an `externref`, therefore, is stored in a side
74/// table (`ExternRefHostDataTable`) and never inside the heap. Only an id
75/// referencing a slot in that table should ever be written into the GC heap.
76///
77/// These constraints give us great amounts of safety compared to working with
78/// raw pointers. The worst that could happen is corruption local to heap and a
79/// panic, or perhaps reading stale heap data from a previous Wasm instance. A
80/// corrupt `GcHeap` can *never* result in the native host's corruption.
81///
82/// The downside is that we are introducing `heap_base + index` computations and
83/// bounds checking to access GC memory, adding performance overhead. This is
84/// deemed to be a worthy trade off. Furthermore, it isn't even a clear cut
85/// performance degradation since this allows us to use 32-bit "pointers",
86/// giving us more compact data representations and the improved cache
87/// utilization that implies.
88pub unsafe trait GcHeap: 'static + Send + Sync {

124    /// Read barrier called every time the runtime clones a GC reference.
125    ///
126    /// Callers should pass a valid `VMGcRef` that belongs to the given
127    /// heap. Failure to do so is memory safe, but may result in general
128    /// failures such as panics or incorrect results.

130
131    /// Write barrier called whenever the runtime is nulling out a GC reference.
132    ///
133    /// Default implemented in terms of the `write_gc_ref` barrier.
134    ///
135    /// If an `externref` is reclaimed, then its associated entry in the
136    /// `host_data_table` should be removed.
137    ///
138    /// Callers should pass a valid `VMGcRef` that belongs to the given
139    /// heap. Failure to do so is memory safe, but may result in general
140    /// failures such as panics or incorrect results.

142    /// The given `gc_ref` should not be used again.
143    fn drop_gc_ref(&mut self, host_data_table: &mut ExternRefHostDataTable, gc_ref: VMGcRef) {

145        self.write_gc_ref(host_data_table, &mut dest, None);
146    }

153    ///
154    /// If an `externref` is reclaimed, then its associated entry in the
155    /// `host_data_table` should be removed.
156    ///
157    /// Callers should pass a valid `VMGcRef` that belongs to the given heap for

175
176    /// Predicate invoked before calling into or returning to Wasm to determine
177    /// whether we should GC first.
178    ///
179    /// `num_gc_refs` is the number of non-`i31ref` GC references that will be

187    /// layout.
188    ///
189    /// Return values:
190    ///
191    /// * `Ok(Some(_))`: The allocation was successful.
192    ///

362    /// # Safety
363    ///
364    /// The returned pointer, if any, must remain valid as long as `self` is not

373    /// Calling this method unassociates this heap with the store that it has
374    /// been associated with, making it available to be associated with a new
375    /// heap.
376    ///

389/// The layout of a GC-managed object.
390#[derive(Clone, Debug)]
391pub enum GcLayout {
392    /// The layout of a GC-managed array object.
393    #[allow(dead_code)] // Not used yet, but added for completeness.
394    Array(GcArrayLayout),
395
396    /// The layout of a GC-managed struct object.
397    Struct(GcStructLayout),
398}
399
400impl From<GcArrayLayout> for GcLayout {
401    fn from(layout: GcArrayLayout) -> Self {
402        Self::Array(layout)
403    }
404}
405
406impl From<GcStructLayout> for GcLayout {
407    fn from(layout: GcStructLayout) -> Self {
408        Self::Struct(layout)
409    }
410}
411
412impl GcLayout {
413    /// Get the underlying `GcStructLayout`, or panic.
414    #[track_caller]
415    pub fn unwrap_struct(&self) -> &GcStructLayout {
416        match self {
417            Self::Struct(s) => s,
418            _ => panic!("GcLayout::unwrap_struct on non-struct GC layout"),
419        }
420    }
421
422    /// Get the underlying `GcArrayLayout`, or panic.
423    #[track_caller]
424    pub fn unwrap_array(&self) -> &GcArrayLayout {
425        match self {
426            Self::Array(a) => a,
427            _ => panic!("GcLayout::unwrap_array on non-array GC layout"),
428        }
429    }
430}

432/// The layout of a GC-managed array.
433///
434/// This layout is only valid for use with the GC runtime that created it. It is
435/// not valid to use one GC runtime's layout with another GC runtime, doing so
436/// is memory safe but will lead to general incorrectness like panics and wrong
437/// results.
438///
439/// All offsets are from the start of the object; that is, the size of the GC
440/// header (for example) is included in the offset.
441///
442/// All arrays are composed of the generic `VMGcHeader`, followed by
443/// collector-specific fields, followed by the contiguous array elements
444/// themselves. The array elements must be aligned to the element type's natural
445/// alignment.
446#[derive(Clone, Debug)]
447#[allow(dead_code)] // Not used yet, but added for completeness.
448pub struct GcArrayLayout {
449    /// The size of this array object, ignoring its elements.
450    pub size: u32,
451
452    /// The alignment of this array.
453    pub align: u32,
454
455    /// The offset of the array's length.
456    pub length_field_offset: u32,
457
458    /// The offset from where this array's contiguous elements begin.
459    pub elems_offset: u32,
460
461    /// The size and natural alignment of each element in this array.
462    pub elem_size: u32,
463}
464
465impl GcArrayLayout {
466    /// Get the total size of this array for a given length of elements.
467    pub fn size_for_len(&self, len: u32) -> u32 {
468        self.size + len * self.elem_size
469    }
470
471    /// Get the offset of the `i`th element in an array with this layout.
472    #[inline]
473    pub fn elem_offset(&self, i: u32, elem_size: u32) -> u32 {
474        self.elems_offset + i * elem_size
475    }
476}

500/// A list of GC roots.
501///
502/// This is effectively a builder for a `GcRootsIter` that will be given to a GC
503/// heap when it is time to perform garbage collection.
504#[derive(Default)]
505pub struct GcRootsList(Vec<RawGcRoot>);
506
507// Ideally these `*mut`s would be `&mut`s and we wouldn't need as much of this
508// machinery around `GcRootsList`, `RawGcRoot`, `GcRoot`, and `GcRootIter` but
509// if we try that then we run into two different kinds of lifetime issues:
510//
511// 1. When collecting the various roots from a `&mut StoreOpaque`, we borrow
512//    from `self` to push new GC roots onto the roots list. But then we want to
513//    call helper methods like `self.for_each_global(...)`, but we can't because
514//    there are active borrows of `self` preventing it.
515//
516// 2. We want to reuse the roots list and its backing storage across GCs, rather
517//    than reallocate on every GC. But the only place for the roots list to live
518//    such that it is easily reusable across GCs is in the store itself. But the
519//    contents of the roots list (when it is non-empty, during GCs) borrow from
520//    the store, which creates self-references.
521#[derive(Clone, Copy, Debug)]
522enum RawGcRoot {
523    Stack(SendSyncPtr<u32>),
524    NonStack(SendSyncPtr<VMGcRef>),
525}
526
527impl GcRootsList {
528    /// Add a GC root that is inside a Wasm stack frame to this list.
529    #[inline]
530    pub unsafe fn add_wasm_stack_root(&mut self, ptr_to_root: SendSyncPtr<u32>) {
531        log::trace!(
532            "Adding Wasm stack root: {:#p} -> {:#p}",
533            ptr_to_root,
534            VMGcRef::from_raw_u32(*ptr_to_root.as_ref()).unwrap()
535        );
536        debug_assert!(VMGcRef::from_raw_u32(*ptr_to_root.as_ref()).is_some());
537        self.0.push(RawGcRoot::Stack(ptr_to_root));
538    }
539
540    /// Add a GC root to this list.
541    #[inline]
542    pub unsafe fn add_root(&mut self, ptr_to_root: SendSyncPtr<VMGcRef>, why: &str) {
543        log::trace!(
544            "Adding non-stack root: {why}: {:#p}",
545            ptr_to_root.as_ref().unchecked_copy()
546        );
547        self.0.push(RawGcRoot::NonStack(ptr_to_root))
548    }
549
550    /// Get an iterator over all roots in this list.
551    ///
552    /// # Safety
553    ///
554    /// Callers must ensure that all the pointers to GC roots that have been
555    /// added to this list are valid for the duration of the `'a` lifetime.
556    #[inline]
557    pub unsafe fn iter<'a>(&'a mut self) -> GcRootsIter<'a> {
558        GcRootsIter {
559            list: self,
560            index: 0,
561        }
562    }
563
564    /// Is this list empty?
565    pub fn is_empty(&self) -> bool {
566        self.0.is_empty()
567    }
568
569    /// Clear this GC roots list.
570    #[inline]
571    pub fn clear(&mut self) {
572        self.0.clear();
573    }
574}

576/// An iterator over all the roots in a `GcRootsList`.
577pub struct GcRootsIter<'a> {
578    list: &'a mut GcRootsList,
579    index: usize,
580}
581
582impl<'a> Iterator for GcRootsIter<'a> {
583    type Item = GcRoot<'a>;
584
585    #[inline]
586    fn next(&mut self) -> Option<Self::Item> {
587        let root = GcRoot {
588            raw: self.list.0.get(self.index).copied()?,
589            _phantom: marker::PhantomData,
590        };
591        self.index += 1;
592        Some(root)
593    }
594}

596/// A GC root.
597///
598/// This is, effectively, a mutable reference to a `VMGcRef`.
599///
600/// Collector implementations should update the `VMGcRef` if they move the
601/// `VMGcRef`'s referent during the course of a GC.
602#[derive(Debug)]
603pub struct GcRoot<'a> {
604    raw: RawGcRoot,
605    _phantom: marker::PhantomData<&'a mut VMGcRef>,
606}
607
608impl GcRoot<'_> {
609    /// Is this root from inside a Wasm stack frame?
610    #[inline]
611    pub fn is_on_wasm_stack(&self) -> bool {
612        matches!(self.raw, RawGcRoot::Stack(_))
613    }
614
615    /// Get this GC root.
616    ///
617    /// Does NOT run GC barriers.
618    #[inline]
619    pub fn get(&self) -> VMGcRef {
620        match self.raw {
621            RawGcRoot::NonStack(ptr) => unsafe { ptr::read(ptr.as_ptr()) },
622            RawGcRoot::Stack(ptr) => unsafe {
623                let raw: u32 = ptr::read(ptr.as_ptr());
624                VMGcRef::from_raw_u32(raw).expect("non-null")
625            },
626        }
627    }
628
629    /// Set this GC root.
630    ///
631    /// Does NOT run GC barriers.
632    ///
633    /// Collector implementations should use this method to update GC root
634    /// pointers after the collector moves the GC object that the root is
635    /// referencing.
636    pub fn set(&mut self, new_ref: VMGcRef) {
637        match self.raw {
638            RawGcRoot::NonStack(ptr) => unsafe {
639                ptr::write(ptr.as_ptr(), new_ref);
640            },
641            RawGcRoot::Stack(ptr) => unsafe {
642                ptr::write(ptr.as_ptr(), new_ref.as_raw_u32());
643            },
644        }
645    }
646}

648/// A garbage collection process.
649///
650/// Implementations define the `collect_increment` method, and then consumers
651/// can either use
652///
653/// * `GarbageCollection::collect` for synchronous code, or
654///
655/// * `collect_async(Box<dyn GarbageCollection>)` for async code.
656///
657/// When using fuel and/or epochs, consumers can also use `collect_increment`
658/// directly and choose to abandon further execution in this GC's heap's whole
659/// store if the GC is taking too long to complete.
660pub trait GarbageCollection<'a>: Send + Sync {
661    /// Perform an incremental slice of this garbage collection process.
662    ///
663    /// Upon completion of the slice, a `GcProgress` is returned which informs
664    /// the caller whether to continue driving this GC process forward and
665    /// executing more slices (`GcProgress::Continue`) or whether the GC process
666    /// has finished (`GcProgress::Complete`).
667    ///
668    /// The mutator does *not* run in between increments. This method exists
669    /// solely to allow cooperative yielding
670    fn collect_increment(&mut self) -> GcProgress;
671
672    /// Run this GC process to completion.
673    ///
674    /// Keeps calling `collect_increment` in a loop until the GC process is
675    /// complete.
676    fn collect(&mut self) {
677        loop {
678            match self.collect_increment() {
679                GcProgress::Continue => continue,
680                GcProgress::Complete => return,
681            }
682        }
683    }
684}
685
686/// The result of doing an incremental amount of GC.
687pub enum GcProgress {
688    /// There is still more work to do.
689    Continue,
690    /// The GC is complete.
691    Complete,
692}
693
694/// Asynchronously run the given garbage collection process to completion,
695/// cooperatively yielding back to the event loop after each increment of work.
696#[cfg(feature = "async")]
697pub async fn collect_async<'a>(mut collection: Box<dyn GarbageCollection<'a> + 'a>) {
698    loop {
699        match collection.collect_increment() {
700            GcProgress::Continue => crate::runtime::vm::Yield::new().await,
701            GcProgress::Complete => return,
702        }
703    }
704}
