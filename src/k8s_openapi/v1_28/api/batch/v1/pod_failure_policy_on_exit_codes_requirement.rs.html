3/// PodFailurePolicyOnExitCodesRequirement describes the requirement for handling a failed pod based on its container exit codes. In particular, it lookups the .state.terminated.exitCode for each app container and init container status, represented by the .status.containerStatuses and .status.initContainerStatuses fields in the Pod status, respectively. Containers completed with success (exit code 0) are excluded from the requirement check.

7    pub container_name: Option<String>,

9    /// Represents the relationship between the container exit code(s) and the specified values. Containers completed with success (exit code 0) are excluded from the requirement check. Possible values are:
10    ///
11    /// - In: the requirement is satisfied if at least one container exit code
12    ///   (might be multiple if there are multiple containers not restricted

14    /// - NotIn: the requirement is satisfied if at least one container exit code
15    ///   (might be multiple if there are multiple containers not restricted

17    /// Additional values are considered to be added in the future. Clients should react to an unknown operator by assuming the requirement is not satisfied.
